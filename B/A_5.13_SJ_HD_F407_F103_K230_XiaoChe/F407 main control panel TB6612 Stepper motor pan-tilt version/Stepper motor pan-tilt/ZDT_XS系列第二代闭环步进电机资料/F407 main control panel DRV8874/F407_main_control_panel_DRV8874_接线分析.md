# F407 main control panel DRV8874 接线分析

> 整理日期：2026-05-17  
> 分析范围：只看**通过排针 / 排母 / 针座引出来**的接口  
> 暂不考虑：现有业务代码逻辑  
> 主要依据：  
> - `PCB schematic diagram\F407 main control panel.enet`  
> - `code\AAA_F407VET6_5.6_Test\F407VET6_5.1_Test.ioc`

> 说明：本文里的 `pin1/pin2/...` 来自网络表编号；像 `U2` 这种双排针，真正压线前仍建议结合 PCB 丝印上的 `Pin1` 标记再复核一次左右列方向。

---

## 1. 先说结论

这块板的“可直接利用接口”可以分成 5 类：

1. `电源输入/输出`
2. `两路 CAN 控制器引出`
3. `串口 / I2C / 灰度传感器接口`
4. `PWM / 电机驱动控制接口`
5. `下载调试接口`

最重要的结论有 6 条：

1. `CAN.1` 和 `CAN.2` **不是 CAN_H / CAN_L 总线口**，而是 MCU 的 `CAN RX/TX` 裸信号。
2. 如果你要接 ZDT 闭环步进电机，**必须外接 CAN 收发器**，不能把电机的 `CANH/CANL` 直接插上去。
3. 板上逻辑接口默认都是 `3.3V` 逻辑，**不能直接接 5V TTL 或 RS232 电平**。
4. `U2 / CN2 / H1 / H2 / H4` 这几组很多都是**纯信号口**，有些甚至**没有 GND 和电源脚**，外接时必须另找共地。
5. `视觉信号out`、`灰度-串口-IN`、`预留串口` 这些是**最容易直接用起来**的接口。
6. 如果目标还是你前面那套“双轴 ZDT 云台”，这块板上**最值得用的是 `CAN.1` 或 `CAN.2` + 外部 CAN 收发器**，而不是 `U2` 那套 DRV8874 控制口。

---

## 2. 电源结构

板上电源主链路是：

`CN1 XT30 输入` -> `F1 自恢复保险丝` -> `U7 电源拨动开关` -> `VCC+12V`

然后：

- `VCC+12V` 进入 `U3 DCDC`
- `U3` 输出 `VCC+3.3V`
- 板上所有逻辑接口基本都用 `3.3V`

### 2.1 电源输入口 `CN1`

接口：`XT30`

引脚定义：

- `CN1 pin1` -> `GND`
- `CN1 pin2` -> 输入正极，经过保险丝和开关后才变成板上 `12V`

注意：

- `CN1 pin3/pin4` 在网络表里没有实际网络，基本可以视作机械焊盘或未用
- 这块板是**自供电板**，很多接口不会额外给你 5V

### 2.2 视觉供电输出 `视觉供电out`

接口：`2P XH2.54`

引脚定义：

- `pin1` -> `GND`
- `pin2` -> `VCC+12V`

说明：

- 这是**12V 输出**
- 适合给视觉板或外部小模块供电
- 不适合直接拖大电流电机负载

## 4. 逐个接口怎么接

## 4.1 `CAN.1`

接口类型：`3P XH2.54`

引脚定义：

- `pin1` -> `PD0` -> `CAN1_RX`
- `pin2` -> `PD1` -> `CAN1_TX`
- `pin3` -> `GND`

这个口**不是标准 CAN 总线口**，而是 MCU 到收发器之间的数字侧信号。

正确接法应该是：

- `CAN.1 pin1` -> 外部 CAN 收发器 `RXD`
- `CAN.1 pin2` -> 外部 CAN 收发器 `TXD`
- `CAN.1 pin3` -> 外部 CAN 收发器 `GND`
- 外部收发器再输出 `CANH / CANL` -> 接电机 CAN 总线

总线侧还要注意：

- `CANH/CANL` 是差分总线，不是单点串口
- 总线两端要按实际拓扑考虑 `120Ω` 终端
- 两台电机挂同一总线时，建议采用总线型布线，不要乱分叉太长

不能这么接：

- 不能把电机的 `CANH` 直接接 `pin1`
- 不能把电机的 `CANL` 直接接 `pin2`

适用场景：

- 双轴 ZDT 云台主控制总线，优先推荐

---

## 4.2 `CAN.2`

接口类型：`3P XH2.54`

引脚定义：

- `pin1` -> `PB12` -> `CAN2_RX`
- `pin2` -> `PB6` -> `CAN2_TX`
- `pin3` -> `GND`

性质和 `CAN.1` 完全一样，也是**裸 CAN 控制器引脚**。

推荐用途：

- 第二路独立 CAN
- 备用总线
- 或者以后做调试隔离

如果你只是控制两台 ZDT 电机：

- 一般 `CAN.1` 一路就够
- 两台电机挂同一条 `CANH/CANL` 总线即可

---

## 4.3 `视觉信号out`

接口类型：`4P 2.50`

引脚定义：

- `pin1` -> `GND`
- `pin2` -> `PA1` -> `UART4_RX`
- `pin3` -> `PA0` -> `UART4_TX`
- `pin4` -> `NC`

接外部视觉主控时按交叉法接：

- 外部设备 `TX` -> 本板 `pin2`
- 外部设备 `RX` -> 本板 `pin3`
- 外部设备 `GND` -> 本板 `pin1`

注意：

- 这是 `3.3V TTL UART`
- 不是 `RS232`
- `pin4` 目前没接东西

这个接口很适合：

- 视觉主板和 F407 之间的高速命令通信
- 比如传目标角度、跟踪偏差、状态字

---

## 4.4 `预留串口`

接口类型：`4P 2.50`

引脚定义：

- `pin1` -> `3.3V`
- `pin2` -> `GND`
- `pin3` -> `PC12` -> `UART5_TX`
- `pin4` -> `PD2` -> `UART5_RX`

推荐接法：

- 外设 `RX` -> `pin3`
- 外设 `TX` -> `pin4`
- 外设 `GND` -> `pin2`
- 如果外设需要少量 `3.3V` 逻辑供电，可以取 `pin1`

适合做：

- 备用调试串口
- 蓝牙/串口屏/上位机调试口
- 与另一块小控制板通信

---

## 4.5 `灰度-串口-IN`

接口类型：`4P 2.50`

引脚定义：

- `pin1` -> `3.3V`
- `pin2` -> `GND`
- `pin3` -> `PC7` -> `USART6_RX`
- `pin4` -> `PC6` -> `USART6_TX`

这也是标准 `3.3V TTL UART`。

接法：

- 外设 `TX` -> `pin3`
- 外设 `RX` -> `pin4`
- 电源按需要接 `pin1/pin2`

---

## 4.6 `灰度-i2c-IN`

接口类型：`4P 2.50`

引脚定义：

- `pin1` -> `3.3V`
- `pin2` -> `GND`
- `pin3` -> `PA8` -> `I2C3_SCL`
- `pin4` -> `PC9` -> `I2C3_SDA`

适合接：

- 灰度模块
- 小型 I2C 传感器
- 外部扩展 IO

注意：

- 这是 `3.3V I2C`
- 从网络表看，`PA8/PC9` 这两根线只连到 MCU 和接口，本板**大概率没有额外上拉**
- 所以外接模块最好自带 I2C 上拉，或者你在线束/子板上补上拉

---

## 4.7 `灰度-IN`

接口类型：`10P XH2.54`

引脚定义：

- `pin1` -> `3.3V`
- `pin2` -> `GND`
- `pin3` -> `PE2`
- `pin4` -> `PE3`
- `pin5` -> `PE4`
- `pin6` -> `PE5`
- `pin7` -> `PE6`
- `pin8` -> `PC13`
- `pin9` -> `PC0`
- `pin10` -> `PC1`

从 `.ioc` 看，这 8 根信号线当前都被当作输入使用，名字分别像：

- `Orgial_HD_1` 到 `Orgial_HD_8`

所以它非常像：

- 8 路灰度传感器输入
- 8 路开关量输入
- 8 路限位/触发输入

这组接口好处是：

- 自带 `3.3V`
- 自带 `GND`
- 信号完整

---

## 4.8 `PWM.1` 和 `PWM.2`

接口类型：`2P XH2.54`

`PWM.1`：

- `pin1` -> `PB14` -> `TIM12_CH1`
- `pin2` -> `GND`

`PWM.2`：

- `pin1` -> `PB15` -> `TIM12_CH2`
- `pin2` -> `GND`

这两个接口的特点是：

- 只有 `PWM + GND`
- **没有 5V 电源**

所以如果拿来接舵机：

- 信号线接这里没问题
- 舵机供电必须单独给
- 地线必须和本板共地

---

## 4.9 `U2`：最像 DRV8874 控制排针

接口类型：`2x4 2.54 排针`

引脚定义：

- `pin1` -> `PA2` -> `TIM2_CH3` -> `DJ_A_IN1`
- `pin2` -> `PA3` -> `TIM2_CH4` -> `DJ_A_IN2`
- `pin3` -> `PE9` -> `TIM1_CH1` -> `DJ_B_IN1`
- `pin4` -> `PE11` -> `TIM1_CH2` -> `DJ_B_IN2`
- `pin5` -> `PE13` -> `TIM1_CH3` -> `DJ_C_IN1`
- `pin6` -> `PE14` -> `TIM1_CH4` -> `DJ_C_IN2`
- `pin7` -> `PB3` -> `TIM2_CH2` -> `DJ_D_IN1`
- `pin8` -> `PA15` -> `TIM2_CH1` -> `DJ_D_IN2`

这个接口最像什么？

- 一组给 `DRV8874` 之类双输入驱动芯片的控制排针
- 一共能控制 `A/B/C/D` 四路
- 每路是 `IN1/IN2`

非常适合：

- 外接 4 路 H 桥/电机驱动板
- 做直流电机、摩擦轮、云台小电机、执行器控制

但要注意：

- 这组排针**没有 GND**
- 也没有电机电源
- 外接驱动板时必须再单独补一根共地

如果你是要接 `ZDT` 一体化闭环步进：

- 这组**不是优先方案**
- 因为 ZDT 更适合 `CAN`

---

## 4.10 `CN2`

接口类型：`4P 2.54 排针`

引脚定义：

- `pin1` -> `PB5` -> `TIM3_CH2`
- `pin2` -> `PB4` -> `TIM3_CH1`
- `pin3` -> `PD13` -> `TIM4_CH2`
- `pin4` -> `PD12` -> `TIM4_CH1`

从 `.ioc` 看，这 4 根线当前最像两组编码器输入：

- `PB4/PB5` 一组
- `PD12/PD13` 一组

但要特别注意：

- 这个口**没有 GND**
- 也没有 `3.3V/5V`

所以它不适合直接插“现成传感器模块”，更适合：

- 接外部驱动板上的编码器反馈信号
- 或者接已经共地的另一块板

---

## 4.11 `H1 / H2 / H4`

这三组都是**MCU 直出信号排针**，但板上没有给它们配成完整外设接口。

### `H1`

- `pin1` -> `PC10`
- `pin2` -> `PC11`

当前 `.ioc` 没有配置，基本可当自由 IO 或后续复用。

### `H2`

- `pin1` -> `PB1`
- `pin2` -> `PB2`
- `pin3` -> `PE7`
- `pin4` -> `PE8`

注意：

- `PB2` 带 `BOOT1` 背景，不建议外接强驱动信号

### `H4`

- `pin1` -> `PD10`
- `pin2` -> `PD11`
- `pin3` -> `PD14`
- `pin4` -> `PD15`

这组也都属于自由度较高的直连 IO。

这 3 组统一注意：

- **都没有 GND**
- **都没有电源**
- 只适合接“你完全知道自己在干什么”的外部板

---

## 4.12 `下载`

接口类型：`3P 2.54`

引脚定义：

- `pin1` -> `GND`
- `pin2` -> `PA14` -> `SWCLK`
- `pin3` -> `PA13` -> `SWDIO`

这就是简化版 `SWD` 下载口。

能做什么：

- 接 `ST-Link`
- 烧录和调试程序

要注意：

- 这个口**没有 `NRST`**
- 也没有 `3.3V` 参考电压
- 所以调试时最好让板子自己先上电

---

## 5. 如果目标是接 ZDT 双轴闭环步进，建议怎么接

如果你后面还是要拿这块板去控制：

- `2 台 ZDT 闭环步进电机`
- `视觉追踪双轴云台`

那么推荐接线方案是：

### 方案 A：只用一路 CAN

1. `CN1` 给这块 F407 板供 `12V`
2. 从 `CAN.1` 接一个**外置 3.3V CAN 收发器小板**
3. 收发器输出 `CANH/CANL`
4. 两台 ZDT 电机并到同一条 `CANH/CANL` 总线上
5. 两台电机各自单独接自己的动力电源
6. 视觉主控如果要和 F407 通信，用 `视觉信号out`

这是最顺的方式。

### 方案 B：两路 CAN 分开

如果你想让：

- `Yaw` 一路总线
- `Pitch` 一路总线

也可以：

- `CAN.1` 接第一块收发器
- `CAN.2` 接第二块收发器

但对你当前这类双轴云台，通常没必要，复杂度更高。

---

## 6. 哪些接口不建议你优先碰

如果目标是 ZDT 云台，我不建议优先把精力花在这些接口上：

- `U2`
- `CN2`
- `H1`
- `H2`
- `H4`

原因不是它们不能用，而是：

- 它们更偏本板原始直驱/编码器/预留 IO 架构
- 不适合直接接一体化 `CAN` 步进电机
- 很多还缺 `GND` 或完整电源脚，接线不如 `CAN + UART` 干净

---

## 7. 这块板最值得用的外接接口

如果站在“你现在要尽快做一套能跑的系统”角度，我建议优先只用下面几组：

- `CN1`：板卡主供电
- `CAN.1`：主电机控制总线数字侧
- `视觉信号out`：视觉主控串口
- `视觉供电out`：给视觉主控供 12V
- `预留串口`：调试或扩展

如果要接额外传感器，再考虑：

- `灰度-i2c-IN`
- `灰度-串口-IN`
- `灰度-IN`

---

## 8. 一句话版结论

这块 `F407 main control panel DRV8874` 真正适合你当前双轴 ZDT 云台项目的接法是：

> `CN1` 给板子供电，`CAN.1` 或 `CAN.2` 先接**外部 CAN 收发器**，再去连 ZDT 电机总线；视觉板走 `视觉信号out + 视觉供电out`；其余 `U2/CN2/H1/H2/H4` 更像原始电机驱动/编码器/预留 IO，不是你现在的优先主链路。

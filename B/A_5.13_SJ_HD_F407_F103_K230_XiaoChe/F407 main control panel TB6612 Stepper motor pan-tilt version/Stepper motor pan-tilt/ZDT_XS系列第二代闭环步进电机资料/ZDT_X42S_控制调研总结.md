## 1. 调研结论先看版

### 1.1 这台设备本质上是什么

这不是“裸步进电机”，而是：

- 步进电机本体
- 编码器
- 闭环驱动板
- 电源与控制接口
- 局部参数存储与小屏菜单

集成在一起的闭环执行器。

也就是说，外部控制器不需要自己去做绕组换相，不需要外接传统步进驱动器。外部系统只需要：

1. 给它供电
2. 选择一种控制链路
3. 发脉冲或发通信命令
4. 按需要读取位置、速度、状态、回零结果

---

### 1.2 它支持哪些控制方式

资料确认该产品支持以下控制路径：

- 脉冲控制
- 串口 TTL 自由协议
- RS485 自由协议
- CAN 自由协议
- RS485 + Modbus-RTU
- DMX512

其中，后续开发最常见、最值得优先考虑的是：

- `脉冲控制`：适合 PLC、老运动控制器、简单 MCU
- `TTL/RS485 自由协议`：适合 STM32、Arduino、ESP32、树莓派、PC 上位机
- `RS485 + Modbus-RTU`：适合工业现场、PLC、组态、标准化集成
- `CAN`：适合多电机同步、高实时性、多轴系统

---

### 1.3 最重要的判断：先确认固件类型

这套产品最关键的一个点是：**同一硬件可以运行不同闭环固件**。

当前资料明确提到两类固件：

- `Emm5.0` 固件
- `X` 固件

这两种固件：

- 大量命令相似
- 但运动命令格式并不完全相同
- 速度单位和位置单位也不完全相同
- 支持的控制模式也不完全相同

因此，**任何程序开发前，第一步都应该确认当前电机实际跑的是 Emm 还是 X 固件**。

如果不先确认，最容易出现的问题是：

- 命令格式发错
- 单位理解错
- 读到的位置值解释错
- 以为支持力矩模式，实际上当前固件没有

## 5. 固件体系：Emm 与 X 的差异

---

### 5.1 为什么这个差异特别关键

同样一台电机，如果当前跑的是不同固件：

- 运动命令编码会不同
- 位置单位会不同
- 速度单位会不同
- 可用控制模式会不同
- 某些高级功能是否存在也会不同

因此必须把“控制电机”理解为：

> **先识别固件，再套对应协议。**

---

### 5.2 Emm 固件特点

Emm 固件的特点：

- 更偏向高速位置跟随
- 更适合 3D 打印机、雕刻机、写字机等
- 速度模式和位置模式比较直接
- 没有标准的力矩模式
- 支持快速位置模式
- 支持狂暴模式 `EmmTurbo`

### 5.3 X 固件特点

X 固件的特点：

- 支持力矩模式
- 支持力位混合控制
- 支持标准梯形曲线加减速位置模式
- 支持在全过程限制电流与速度
- 更适合机械臂、夹爪、大负载结构、复杂运动控制

### 5.4 固件差异概括

| 对比项 | Emm 固件 | X 固件 |
| --- | --- | --- |
| 力矩模式 | 不支持 | 支持 |
| 速度模式 | 支持，命令较简单 | 支持，速度单位多为 `0.1RPM` |
| 位置模式 | 用“脉冲数”表达 | 用“角度值”表达 |
| 快速位置模式 | 支持 | 支持快速梯形位置模式 |
| 力位混合 | 无 | 有 |
| 应用偏向 | 3D 打印/雕刻/写字 | 机械臂/夹爪/复杂运动 |

### 5.5 如何识别当前固件

可用方法：

1. 看小屏显示界面
2. 看 LED 表现
3. 用上位机读取版本
4. 发送读取版本命令 `1F`

建议开发时不要只靠“默认值”猜测，最好在代码初始化时直接读取版本信息并记录。

## 6. 接口与接线

---

### 6.1 端子大致含义

控制端常见信号如下：

- `V+`：电源正
- `Gnd`：电源地
- `Com`：脉冲控制公共端
- `En`：使能/复用端
- `Stp`：脉冲输入/复用端
- `Dir`：方向输入/复用端
- `R/A/H`：串口 RX / RS485 A / CAN H / 复用端
- `T/B/L`：串口 TX / RS485 B / CAN L / 复用端

这些端子不是永远固定单一功能，很多端子可以复用为：

- 通信
- 限位输入
- 回零输入
- 到位输出
- 报警输出

所以接线前一定要确认当前参数里的复用模式。

---

### 6.2 脉冲控制接线

资料中的脉冲接线图明确给出了一种常见共阳接法：

- `Gnd -> MCU GND`
- `Com -> 3.3V`
- `En -> MCU GPIO`
- `Stp -> MCU GPIO`
- `Dir -> MCU GPIO`

旧版兼容例程图中给出了：

- `En -> A5`
- `Stp -> A6`
- `Dir -> A7`

这只是示例，不是必须固定用这些口。

### 6.3 脉冲控制的本质

脉冲控制时：

- `Dir` 决定方向
- `Stp` 每跳变一次相当于输入一个步进脉冲
- `En` 决定驱动是否使能

默认 `1.8°` 电机、`16` 细分时：

- `3200` 脉冲 = `1` 圈 = `360°`

换算公式：

- `每圈脉冲数 = 200 * 细分`，适用于 `1.8°`
- `每圈脉冲数 = 400 * 细分`，适用于 `0.9°`

角度换算公式：

```text
角度(°) = 脉冲数 * 360 / 每圈脉冲数
```

### 6.4 脉冲控制的优缺点

优点：

- 接入最简单
- 与 PLC / 传统运动控制器兼容性高
- 不需要理解通信协议

缺点：

- 状态回读能力弱
- 多轴同步能力不如 CAN / 通信协议灵活
- 高级功能要靠参数配置或额外引脚复用实现

---

### 6.5 TTL 串口接线

新版上位机接线图确认：

- `GND -> USB-TTL 的 GND`
- `R/A/H -> USB-TTL 的 TX`
- `T/B/L -> USB-TTL 的 RX`

TTL 模式下必须共地。

默认串口参数通常为：

- 波特率：`115200`
- 数据位：`8`
- 校验位：`None`
- 停止位：`1`

---

### 6.6 RS485 接线

新版接线图确认：

- `R/A/H -> A`
- `T/B/L -> B`

补充注意：

- RS485 是差分传输，通常比 TTL 更抗干扰
- 总线末端建议加 `120Ω` 终端电阻
- 多机总线控制时很适合使用 RS485

---

### 6.7 CAN 接线

新版接线图确认：

- `R/A/H -> CANH`
- `T/B/L -> CANL`

补充注意：

- CAN 使用扩展帧
- 总线末端需要 `120Ω`
- 适合多电机同步和高实时多轴控制

## 7. 端口复用机制

这部分非常重要，因为很多“为什么通信不通 / 为什么脉冲没反应 / 为什么限位没效果”的根本原因，都是端口复用没设对。

### 7.1 脉冲端口复用

手册给出的脉冲端口复用模式包括：

- `PUL_OFF`
- `PUL_ENA`
- `ESI_RCO`
- `pLR_ESI`

含义如下：

| 模式 | 作用 |
| --- | --- |
| `PUL_OFF` | 关闭脉冲端口 |
| `PUL_ENA` | 正常脉冲控制 |
| `ESI_RCO` | `En` 作为回零限位输入，`Dir` 作为到位输出 |
| `pLR_ESI` | `En/Stp` 作为左右限位输入，`Dir` 作为到位输出 |

### 7.2 通信端口复用

手册给出的通信端口复用模式包括：

- `RxTx_OFF`
- `ESI_ALO`
- `UART_FUN`
- `CAN1_MAP`
- `uLR_ESI`

含义如下：

| 模式 | 作用 |
| --- | --- |
| `RxTx_OFF` | 关闭通信端口 |
| `UART_FUN` | 串口 TTL / RS232 / RS485 通信 |
| `CAN1_MAP` | CAN 通信 |
| `ESI_ALO` | `R/A/H` 作限位回零输入，`T/B/L` 作报警输出 |
| `uLR_ESI` | `R/A/H` 与 `T/B/L` 复用为左右限位输入 |

## 8. 控制前推荐的标准流程

建议把第一次上电调试流程固定成下面这样：

1. 确认供电与电压版本
2. 确认当前固件类型
3. 确认电机类型是 `0.9°` 还是 `1.8°`
4. 做一次编码器校准
5. 设定控制方式与端口复用模式
6. 设定通信参数或细分参数
7. 先做最小动作测试
8. 再做位置/速度读取
9. 再做回零、限位、同步运动等高级功能

推荐的“最小动作测试”顺序：

1. `使能`
2. `低速速度模式`
3. `停止`
4. `读取实时速度`
5. `读取实时位置`
6. `简单位置模式`
7. `读取状态标志`

## 9. 校准、方向、电机类型、控制模式

### 9.1 编码器校准

编码器校准命令：

```text
Addr 06 45 6B
```

作用：

- 做线性化校准
- 提高角度精度

如果不校准，手册明确提示会有较大的角度误差。

### 9.2 电机类型

支持：

- `0.9°`
- `1.8°`

如果电机类型设错，会直接导致：

- 脉冲角度换算错误
- 位置命令与实际运动不一致

### 9.3 正方向

电机正方向可以通过参数修改，也可以通过对调相线改变，但软件优先建议用参数法处理。

### 9.4 开环 / 闭环模式

支持：

- 开环模式
- 闭环 FOC 模式

通常开发时应优先用闭环模式，除非你有明确原因需要切换到开环。

## 10. 通信协议总览

---

### 10.1 自由协议串口格式

TTL / RS485 自由协议默认格式：

```text
地址 + 功能码 + 命令数据 + 校验码
```

默认校验码通常为：

```text
0x6B
```

可切换的校验/协议方式包括：

- 固定 `0x6B`
- `XOR`
- `CRC-8`
- `Modbus`
- `DMX512`

### 10.2 常见返回码

| 返回码 | 含义 |
| --- | --- |
| `02` | 命令已正确接收 |
| `12` | 触发回零时当前已在零点或限位已触发 |
| `9F` | 动作执行完成 |
| `E2` | 参数错误或条件不满足 |
| `EE` | 命令格式错误 |

### 10.3 CAN 格式

CAN 使用扩展帧。

发送格式不是把串口整帧原样搬过去，而是：

- 扩展帧 ID：`(Addr << 8) | Packet`
- 数据区：放功能码和数据

对于大于 8 字节的命令，需要分包。

### 10.4 Modbus-RTU 格式

Modbus 版本使用 RS485 链路。

使用前必须先把参数里的 `Checksum/通讯协议` 改成 `Modbus`。

手册说明：

- `X42` 只能用 `04H` 读取
- `X42S / Y42` 支持 `03H` 或 `04H` 读取

写命令常见形式：

- `06H`：写单寄存器
- `10H`：写多寄存器

一个特别要注意的点：

- `X` 固件下 `32bit` 数据在 Modbus 中“先传低字，再传高字”

## 11. 最常用的自由协议命令

下面按“日常最常用”的优先级整理。

---

### 11.1 通用动作命令

| 功能 | 命令 |
| --- | --- |
| 编码器校准 | `Addr 06 45 6B` |
| 重启电机 | `Addr 08 97 6B` |
| 将当前位置清零 | `Addr 0A 6D 6B` |
| 解除堵转/过热/过流保护 | `Addr 0E 52 6B` |
| 恢复出厂设置 | `Addr 0F 5F 6B` |
| 电机使能控制 | `Addr F3 AB state sync 6B` |
| 立即停止 | `Addr FE 98 sync 6B` |
| 触发同步运动 | `Addr FF 66 6B` |

备注：

- `state`：`00` 不使能，`01` 使能
- `sync`：`00` 立即执行，`01` 先缓存

---

### 11.2 Emm 固件常用运动命令

#### 速度模式

```text
Addr F6 dir vel_hi vel_lo acc sync 6B
```

参数含义：

- `dir`：`00` CW，`01` CCW
- `vel`：单位 `RPM`
- `acc`：`0-255`
- `sync`：同步标志

示例：

```text
01 F6 01 05 DC 0A 00 6B
```

含义：

- 地址 1
- 逆时针
- 1500RPM
- 加速度档位 10
- 立即执行

#### 位置模式

```text
Addr FD dir vel_hi vel_lo acc pulse[4] mode sync 6B
```

参数含义：

- `vel`：单位 `RPM`
- `acc`：`0-255`
- `pulse[4]`：脉冲数
- `mode`：
  - `00` 相对上一输入目标位置
  - `01` 绝对坐标零点
  - `02` 相对当前实时位置

示例：

```text
01 FD 01 05 DC 00 00 00 7D 00 00 00 6B
```

含义：

- 地址 1
- 逆时针
- 1500RPM
- 加速度 0
- 相对运动 32000 脉冲

#### 快速位置模式

先设参数：

```text
Addr F1 vel_hi vel_lo acc mode sync 6B
```

再单独发位置：

```text
Addr FC signed_pulse[4] 6B
```

适合：

- 二维云台
- 高频更新目标位置
- 固定速度和加速度的场景

---

### 11.3 X 固件常用运动命令

#### 力矩模式

```text
Addr F5 sign ramp_hi ramp_lo cur_hi cur_lo sync 6B
```

#### 力矩模式限速

```text
Addr C5 sign ramp_hi ramp_lo cur_hi cur_lo sync vmax_hi vmax_lo 6B
```

单位：

- 斜率：`mA/s`
- 电流：`mA`
- 最大速度：`0.1RPM`

#### 速度模式

```text
Addr F6 dir acc_hi acc_lo vel_hi vel_lo sync 6B
```

单位：

- 加速度：`RPM/s`
- 速度：`0.1RPM`

#### 速度模式限电流

```text
Addr C6 dir acc_hi acc_lo vel_hi vel_lo sync cur_hi cur_lo 6B
```

#### 直通限速位置模式

```text
Addr FB dir vel_hi vel_lo pos[4] mode sync 6B
```

单位：

- 速度：`0.1RPM`
- 位置：`0.1°`

#### 直通限速位置模式限电流

```text
Addr CB dir vel_hi vel_lo pos[4] mode sync cur_hi cur_lo 6B
```

#### 梯形曲线位置模式

```text
Addr FD dir acc_hi acc_lo dec_hi dec_lo vel_hi vel_lo pos[4] mode sync 6B
```

单位：

- 加速加速度：`RPM/s`
- 减速加速度：`RPM/s`
- 最大速度：`0.1RPM`
- 位置：`0.1°`

#### 梯形曲线位置模式限电流

```text
Addr CD dir acc_hi acc_lo dec_hi dec_lo vel_hi vel_lo pos[4] mode sync cur_hi cur_lo 6B
```

#### 快速梯形位置模式

先设参数：

```text
Addr F1 acc dec vmax mode sync max_cur 6B
```

再发位置：

```text
Addr FC signed_pos[4] 6B
```

适合快速多次刷新位置目标。

## 12. 单位与数值解释：这是最容易出错的地方

---

### 12.1 Emm 固件

#### 位置命令

- 位置模式命令中发的是“脉冲数”
- 不是角度值

换算公式：

```text
角度 = 脉冲数 * 360 / 每圈脉冲数
```

默认 `1.8° + 16细分` 时：

```text
每圈脉冲数 = 3200
```

#### 读取实时位置 / 目标位置 / 误差

Emm 固件读取值常见解释为：

```text
0 - 65535 对应单圈 0 - 360°
角度 = 数值 * 360 / 65536
```

#### 读取实时转速

- 单位通常是 `RPM`

---

### 12.2 X 固件

#### 位置命令

- 位置模式通常按 `0.1°` 输入

例如：

- 发送 `36000` 表示 `3600.0°`

若启用了“继续缩小 10 倍输入”：

- 最小可到 `0.01°`

#### 读取实时位置

```text
角度 = 返回值 / 10
```

#### 读取位置误差

```text
误差角度 = 返回值 / 100
```

#### 读取实时转速

```text
转速 = 返回值 / 10 RPM
```

## 13. 回零系统

这台产品的回零能力很强，是它区别于很多普通一体式驱动步进的重要特性。

---

### 13.1 支持的回零模式

手册中列出的回零模式包括：

| 模式值 | 含义 |
| --- | --- |
| `00` | 单圈就近回零 |
| `01` | 单圈方向回零 |
| `02` | 多圈无限位碰撞回零 |
| `03` | 多圈限位开关回零 |
| `04` | 回到绝对位置坐标零点 |
| `05` | 回到上次掉电位置角度 |

---

### 13.2 回零相关命令

| 功能 | 命令 |
| --- | --- |
| 设置单圈零点位置 | `Addr 93 88 save 6B` |
| 触发回零 | `Addr 9A mode sync 6B` |
| 强制中断回零 | `Addr 9C 48 6B` |
| 读取回零状态标志 | `Addr 3B 6B` |
| 读取回零参数 | `Addr 22 6B` |
| 修改回零参数 | `Addr 4C AE ... 6B` |

---

### 13.3 无限位碰撞回零原理

碰撞回零的检测逻辑本质上与堵转检测相似：

满足以下三个条件后判定碰撞成功：

1. 实时转速低于阈值
2. 实时相电流高于阈值
3. 上述状态持续时间超过阈值

这意味着实际调试时通常要：

1. 让电机以计划回零速度运行
2. 读取运行中的相电流
3. 再把“碰撞检测电流阈值”设得略高于正常运行电流

---

### 13.4 回零状态标志

`3B` 返回的回零状态标志可用来判断：

- 编码器是否正常
- 是否已校准
- 当前是否正在回零
- 是否回零失败
- 是否触发过热/过流

程序里非常适合做状态机判断。

## 14. 限位、到位输出、报警输出

---

### 14.1 左右限位

左右限位并不是单独专门的引脚，而是通过端口复用实现。

常见映射：

- 脉冲端口复用后：
  - `En` 作为左限位
  - `Stp` 作为右限位
- 通信端口复用后：
  - `R/A/H` 作为左限位
  - `T/B/L` 作为右限位

### 14.2 到位判断

电机到位的判据：

```text
目标位置 - 实时位置 < 位置到达窗口
```

默认位置到达窗口：

- `0.8°`

### 14.3 到位输出

若脉冲端口复用为 `ESI_RCO`：

- `Dir` 引脚可作为到位输出

电机到位后：

- `Dir` 输出高电平

未到位时：

- `Dir` 输出低电平

### 14.4 到位返回

若控制命令应答方式设为：

- `Reached`
- `Both`
- `Other`

则位置到位后，电机会主动返回：

```text
地址 + 功能码 + 9F + 校验
```

### 14.5 报警输出

若通信端口复用为 `ESI_ALO`：

- `R/A/H` 用于限位输入
- `T/B/L` 用于报警输出

发生堵转等异常时会输出报警电平。

## 15. 多机通信、多轴同步、多电机命令

这部分是本产品很强的能力之一。

---

### 15.1 多机通信

多个电机可以挂在同一条总线上：

- TTL
- RS485
- CAN

但最推荐的多机总线实际一般是：

- `RS485`
- `CAN`

### 15.2 三种同步思路

手册明确给出了三种同步方法：

1. `广播地址 0`
2. `多电机命令 AA`
3. `同步标志 + 触发同步命令 FF`

#### 方法 1：广播地址 0

适合：

- 所有电机执行同一条命令

#### 方法 2：多电机命令 `AA`

适合：

- 不同电机执行不同命令
- 希望一条大命令尽量快地下发到总线

注意：

- Modbus-RTU 暂不支持多电机命令

#### 方法 3：先缓存再 `FF 66`

做法：

1. 给各电机分别发送带 `sync=01` 的命令
2. 电机先缓存，不立刻动作
3. 最后广播发送 `00 FF 66 6B`

这样多个电机会同时起动。

---

## 16. 定时返回信息：高频监控时比轮询更好用

手册新增了“定时返回信息命令”，这是一个非常实用的功能。

命令格式：

```text
Addr 11 18 info_code time_hi time_lo 6B
```

作用：

- 让电机定时主动回传某类状态
- 不需要主机不停轮询

适合：

- 实时监控速度
- 实时监控位置
- 做轻量级闭环上层监视
- 提高多轴系统通信效率

例如：

- 每 `1ms` 返回一次实时位置
- 每 `10ms` 返回一次状态标志

若时间设为 `0000`，则停止定时返回。

## 17. 最值得读取的状态量

开发时最推荐优先接入以下读取命令：

| 功能 | 命令 | 用途 |
| --- | --- | --- |
| 版本 | `1F` | 确认固件与硬件 |
| 总线电压 | `24` | 监控供电 |
| 总线电流 | `26` | 看负载趋势 |
| 相电流 | `27` | 看真实驱动力 / 碰撞回零调参 |
| 编码器值 | `31` | 看单圈角度反馈 |
| 输入脉冲数 | `32` | 脉冲控制时核对输入 |
| 目标位置 | `33` | 看最终目标 |
| 实时设定目标位置 | `34` | 看位置命令更新后的目标 |
| 实时转速 | `35` | 监控运行状态 |
| 实时位置 | `36` | 核心反馈 |
| 位置误差 | `37` | 判断跟随质量 |
| 电机状态标志 | `3A` | 判断使能、到位、堵转、限位 |
| 回零状态标志 | `3B` | 判断回零是否完成 |
| 回零 + 电机状态 | `3C` | 一次读两类状态 |
| IO 电平状态 | `3D` | 调试引脚复用/限位/到位输出 |

## 18. 电机状态标志怎么用

`3A` 返回的电机状态标志是最值得程序接入的状态字之一。

能反映：

- 是否使能
- 是否到位
- 是否发生堵转
- 是否触发堵转保护
- 左右限位引脚状态
- 掉电标志

建议在程序中把它解析成位字段结构，避免后续反复写位运算。

## 19. 参数配置里最常改的几项

开发与调试时，最常修改的不是 PID，而是下面这些基础参数：

### 19.1 地址

命令：

```text
Addr AE 4B save new_addr 6B
```

用于多机总线时必须先规划地址。

### 19.2 细分

命令：

```text
Addr 84 8A save microstep 6B
```

注意：

- `00` 表示 `256` 细分
- `01-FF` 表示 `1-255` 细分

### 19.3 控制模式

命令：

```text
Addr 46 A6 save ctrl_mode 6B
```

- `00`：开环
- `01`：闭环

### 19.4 固件类型

命令：

```text
Addr D5 69 save fw_type 6B
```

手册中给出：

- `00`：X 固件
- `01`：Emm 固件
- `02`：Emm 狂暴模式

注意：

- 修改固件类型前尽量保证电机停止

### 19.5 最大电流

命令：

```text
Addr 45 66 save max_cur 6B
```

解释差异：

- Emm：表示堵转最大电流
- X：表示整个正常运行过程中都不会超过此值

### 19.6 PID 参数

命令：

```text
Addr 21 6B        ; 读 PID
Addr 4A C3 ...    ; 写 PID
```

不建议一开始就改 PID。

正确顺序应该是：

1. 先把电机类型、细分、方向、回零、限位、电流上限调对
2. 跑通基础运动
3. 再看是否真的需要调整 PID

## 20. Modbus-RTU 的实际定位

如果你的上位系统天然偏工业风格，例如：

- PLC
- 触摸屏
- 工业网关
- SCADA
- 标准仪表系统

那最推荐的控制方式通常不是自由协议，而是 `RS485 + Modbus-RTU`。

原因：

- 集成成本低
- 标准化程度高
- 多机管理清晰
- 现场工具多

但要注意两点：

1. 必须先把电机通信协议参数改成 `Modbus`
2. `X` 固件下某些 `32bit` 数据在寄存器顺序上和直觉不同

优先顺序：

推荐：

### 22.3 如果你要读反馈、做复杂流程

推荐：

- `TTL/RS485 自由协议`

适合：

- STM32
- Arduino
- ESP32
- 树莓派
- Python 上位机

### 22.4 如果你是工业项目

推荐：

- `RS485 + Modbus-RTU`

### 22.5 如果你是多轴同步项目

推荐：

- `CAN`
- 或自由协议中的 `AA / FF` 机制

### 22.6 如果你要夹爪、力控、限流位置控制

推荐：

- 切到 `X` 固件

## 23. 实际开发时最容易踩的坑

### 23.1 不先确认固件类型

后果：

- 命令格式发错
- 单位理解错

### 23.2 电机类型设错

后果：

- 位置和角度完全不准

### 23.3 端口被复用后还按原功能接线

后果：

- 串口不通
- 脉冲没反应
- 限位没效果

### 23.4 忘了校准编码器

后果：

- 位置误差大

### 23.5 旧版兼容图与新版手册混用

后果：

- 电压、接口、默认值理解错

### 23.6 RS485 / CAN 不做终端匹配

后果：

- 长线或多节点时容易通信异常

### 23.7 Modbus 没切协议模式

后果：

- 明明线对了，但怎么都读不到

### 23.8 一上来就改 PID

后果：

- 问题没定位，反而引入新变量

## 25. 后续代码开发建议

如果后面要自己写协议库，建议结构如下：

### 25.1 分层

1. `transport` 层
   - UART / RS485 / CAN / Modbus
2. `packet` 层
   - 负责拼包、拆包、校验
3. `device` 层
   - `enable()`
   - `set_speed()`
   - `set_position()`
   - `read_position()`
   - `home()`
4. `app` 层
   - 业务逻辑

### 25.2 先把固件类型抽象出来

建议代码里明确区分：

- `FirmwareEmm`
- `FirmwareX`

不要把两种固件强行揉成一套完全相同的接口，否则后面单位和模式会很乱。

### 25.3 命令单位要统一

对上层程序最好统一成：

- 角度：`degree`
- 速度：`rpm`
- 电流：`mA`

然后在协议层内部再做换算：

- Emm：角度转脉冲
- X：角度乘 `10` 或 `100`

### 25.4 状态字解析一定要封装

至少建议封装：

- 电机状态标志 `3A`
- 回零状态标志 `3B`
- IO 状态 `3D`

这样后续业务层可以直接写：

```text
if status.enabled
if status.position_reached
if home_status.homing_failed
```

而不是满代码位运算。

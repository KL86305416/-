# F407 DRV8874 Emm Stepper Baud Fix

- Date: `2026-05-26`
- Status: `implemented`
- Target project:
  - `F407 main control panel DRV8874\code\AAA_F407VET6_5.6_Test`
- Trigger:
  - OpenMV vision data was confirmed to reach the F407 side.
  - The two X42S motors were confirmed to run `Emm` firmware over UART, but their configured baud rate is `256000`, not `115200`.
  - The current F407 firmware had both motor links still configured for `115200`, which would prevent all motor responses and motion.
- Main changes:
  - Updated `UART5` baud rate from `115200` to `256000`.
  - Updated `USART6` baud rate from `115200` to `256000`.
  - Synced the same values into the CubeMX `.ioc` file to reduce the risk of later regeneration reverting the motor baud rate.
- Scope note:
  - `UART4` for the OpenMV vision link remains at `115200`.
  - No vision protocol or motion-control logic was changed in this step.
- Verification:
  - Pre-edit backup created at:
    - `backups/2026-05-26_f407_drv8874_emm_256000_baud_fix`
  - A host-side static validation pass should be run after patching.
- Runtime expectation:
  - If the motors are truly on `Emm` firmware and `UART_FUN` mode, this change removes the strongest known transport mismatch.
  - If the motors still do not move after this fix, the next suspects are:
    - UART wiring polarity
    - motor address mismatch
    - communication port mode not set to `UART_FUN`
    - lack of motor feedback parsing preventing precise diagnosis

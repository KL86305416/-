## 3. `.gitignore` 策略

本轮 `.gitignore` 采取的是“只忽略明显生成物，不忽略项目过程资料”的策略。

当前忽略范围主要包括：

- `Keil/MDK` 的用户本地文件
- 常见编译产物，如：
  - `*.o`
  - `*.d`
  - `*.crf`
  - `*.axf`
  - `*.hex`
  - `*.map`
  - `*.lst`
- 常见构建输出目录，如：
  - `Debug`
  - `Release`
  - `Listings`
  - `Objects`

这样做的原因是：

- 不把纯生成垃圾卷进仓库
- 保留源代码、原理图、网表、开发日志、阶段摘要和备份记录的可跟踪性

另外，以下当前为空的目录已补入 `.gitkeep`：

- `K230 Visual module(SJ)`
- `reference_library`
- `temp_transfer`

这样后续即使这些目录暂时没有实际文件，Git 里仍能保留工作区骨架。

## 4. 本轮刻意没有做的事

本轮没有自动做：

- `git add .`
- 初始提交
- 远程仓库绑定

原因很简单：

- “初始化为 Git 仓库”和“立刻形成第一版提交”是两个不同动作
- 先把仓库壳建立起来，再由后续阶段决定首个提交边界，会更稳妥

# F407 电机驱动拓扑与 PWM 映射记录

- 日期：2026-05-14
- 工作区：`D:\A_5.13_SJ_HD_F407_F103_K230_XiaoChe`
- 范围：`F407 main control panel`
- 目的：把当前主控侧电机驱动方案、8 路 PWM 分配和与舵机/编码器的边界正式厘清

## 1. 当前确认的驱动方案

根据最新口头确认，`F407` 主控部分实际准备了两套电机驱动思路：

1. `DRV8874`
   - 两路 `PWM` 驱动一路电机
   - 也就是每个电机使用 `IN1 + IN2` 两个 PWM 输入
2. `TB6612`
   - 两路 `PWM` + 五个 `IO` 驱动两路电机

当前实际采用的是：
- `DRV8874`

这意味着当前软件建模应该优先围绕：
- `4` 路电机
- 每路电机 `2` 个 PWM 输入
- 共 `8` 路 PWM 控制线

## 2. 当前 8 路电机 PWM 控制线

结合 `main.h`、`.ioc` 和 `tim.c`，当前已经能确认这 8 路电机 PWM 线就是下面这组 `DJ_*_IN*`：

| 电机组 | 控制线 1 | 控制线 2 | MCU 引脚 | 定时器通道 |
| --- | --- | --- | --- | --- |
| `DJ_A` | `DJ_A_IN1` | `DJ_A_IN2` | `PA2 / PA3` | `TIM2_CH3 / TIM2_CH4` |
| `DJ_B` | `DJ_B_IN1` | `DJ_B_IN2` | `PE9 / PE11` | `TIM1_CH1 / TIM1_CH2` |
| `DJ_C` | `DJ_C_IN1` | `DJ_C_IN2` | `PE13 / PE14` | `TIM1_CH3 / TIM1_CH4` |
| `DJ_D` | `DJ_D_IN1` | `DJ_D_IN2` | `PB3 / PA15` | `TIM2_CH2 / TIM2_CH1` |

这 8 路恰好对应：
- `4` 路电机
- 每路双 PWM
- 与当前 `DRV8874` 方案完全一致

## 3. 当前舵机 PWM 与电机 PWM 的边界

需要特别分开的一点是：

- `PB14 / PB15`
- `TIM12_CH1 / TIM12_CH2`
- `PWM_DuoJi_A / PWM_DuoJi_B`

这两路是舵机 PWM，不是上面那 8 路电机 PWM。

同时，网表中明确能看到两个外部两针接口：
- `PWM.1`
- `PWM.2`

其中：
- `PWM.1` 的信号网是 `B14`
- `PWM.2` 的信号网是对应另一条舵机 PWM 网

因此当前可以明确：
- `PWM.1 / PWM.2` 属于舵机输出口
- `DJ_A/B/C/D_IN1/IN2` 才是当前电机驱动相关的 8 路 PWM 控制线

## 4. 当前编码器反馈口

主控侧当前还已经配好了两组编码器输入：

| 编码器组 | MCU 引脚 | 定时器 |
| --- | --- | --- |
| `DJ_BM_A_1 / DJ_BM_A_2` | `PD12 / PD13` | `TIM4 Encoder` |
| `DJ_BM_B_1 / DJ_BM_B_2` | `PB4 / PB5` | `TIM3 Encoder` |

这说明当前工程不仅准备了电机驱动口，也已经预留了速度反馈口。

## 5. 当前定时器分配状态

### 5.1 电机 PWM

- `TIM1`
  - `CH1 -> PE9 -> DJ_B_IN1`
  - `CH2 -> PE11 -> DJ_B_IN2`
  - `CH3 -> PE13 -> DJ_C_IN1`
  - `CH4 -> PE14 -> DJ_C_IN2`

- `TIM2`
  - `CH1 -> PA15 -> DJ_D_IN2`
  - `CH2 -> PB3 -> DJ_D_IN1`
  - `CH3 -> PA2 -> DJ_A_IN1`
  - `CH4 -> PA3 -> DJ_A_IN2`

### 5.2 舵机 PWM

- `TIM12`
  - `CH1 -> PB14 -> PWM_DuoJi_A`
  - `CH2 -> PB15 -> PWM_DuoJi_B`

## 6. 当前一个很重要的代码风险点

这轮核对里发现了一个必须提前记下来的问题：

- `TIM1.Init.Period = 65535`
- `TIM2.Init.Period = 4294967295`
- `TIM12.Init.Period = 65535`

其中最需要注意的是：
- `TIM2` 当前周期值是 `0xFFFFFFFF`

这说明虽然 `TIM2` 的四个通道已经被配成 PWM 模式，但它现在的基础计数周期明显不适合直接拿来驱电机。

换句话说：
- “PWM 通道已经分配好”
不等于
- “电机 PWM 已经处于可直接上车的工作频率”

## 7. 对当前主控软件建模的含义

基于上面的确认，当前主控侧最合理的软件抽象应当是：

1. 先按 `DRV8874` 模式建一个 `4` 电机抽象层
2. 每个电机暴露：
   - `IN1 PWM`
   - `IN2 PWM`
3. 用“有符号速度/占空比”在软件层做方向映射
4. 把 `TB6612` 保留为后续可切换方案，而不是当前第一优先级

## 8. 当前最自然的下一步

在当前阶段，最自然的下一步不是直接写上层行为逻辑，而是：

1. 先统一 `TIM1 / TIM2` 的电机 PWM 基础参数
2. 新建一个主控侧 `motor_drive` 层
3. 先只支持当前实际使用的 `DRV8874` 双 PWM 模式
4. 等灰度短帧接收稳定后，再把 `TURN` 逐步接到电机控制层

这样推进，系统边界会比较干净，也更符合当前硬件真实使用方式。

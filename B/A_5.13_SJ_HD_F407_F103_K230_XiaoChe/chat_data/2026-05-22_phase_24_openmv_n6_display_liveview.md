# 阶段摘要 24：OpenMV N6 官方屏幕实时取流显示
- 日期：`2026-05-22`
- 阶段编号：`24`
- 主题：`openmv_n6_display_liveview`
- 范围：`OpenMV Visual module(SJ)\code / backups / dev_logs / chat_data`

## 1. 这一阶段为什么开始
在确认 OpenMV N6 上电问题后，当前工作目标转到了应用层：

- 不再是继续排电源
- 而是先把摄像头实时画面显示到官方屏幕背板上

## 2. 这一阶段做了什么
这一阶段在 `OpenMV Visual module(SJ)\code` 下第一次落入了可直接上传到板子的运行脚本：

- `main.py`
- `README.md`

脚本使用官方 OpenMV 的 `csi` 与 `display.SPIDisplay` 路径实现：
- 实时抓帧
- 屏幕显示
- 保持宽高比缩放
- 左上角叠加 `FPS`

## 3. 这一阶段得到的直接结果
当前 OpenMV N6 这条线已经有了一个明确的最小运行入口：

1. 板上摄像头采集图像
2. 官方 SPI 屏显示图像
3. 左上角显示当前 FPS

同时脚本兼容两种官方屏幕类型，通过改一行 `DISPLAY_KIND` 即可切换。

## 4. 当前工程状态
这轮脚本已完成本地语法校验，通过：
- `python -m py_compile`

但尚未在真实 `N6 + 官方屏` 组合上完成实机验证。

## 5. 这一阶段留下的下一步
下一步最自然的是上板联调，确认三件事：

1. 当前 `DISPLAY_KIND` 是否与你手上的官方屏型号一致
2. 屏幕驱动 API 与板上固件版本是否完全匹配
3. 实时显示效果是否需要进一步改成铺满、裁剪或更醒目的 FPS 样式

这一阶段的意义是：OpenMV N6 这条线已经从“只有资料整理”，推进到了“开始有实际运行代码可上板测试”。

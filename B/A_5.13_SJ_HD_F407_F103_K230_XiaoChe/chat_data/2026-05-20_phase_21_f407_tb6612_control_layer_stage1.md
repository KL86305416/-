# 阶段摘要 21：F407 TB6612 主线控制层阶段一
- 日期：`2026-05-20`
- 阶段编号：`21`
- 主题：`f407_tb6612_control_layer_stage1`
- 范围：`F407 main control panel TB6612 / backups / dev_logs / chat_data`

## 1. 这一阶段为什么开始
前一阶段完成的是目录重构与仓库整理，但主线 `F407/TB6612` 工程本身仍停留在“已经能收 F103 灰度短帧、已经能在 OLED 上显示、已经有 TB6612 电机底层初始化”的状态。

真正缺失的部分，是主控把这些能力串起来的第一层控制入口。也就是说：
- 主控能看到灰度数据
- 主控能初始化电机
- 但主控还不能安全、直接、独立地验证电机输出

因此这一阶段的目标不是立刻自动寻迹，而是先给主线补上一个最小可用的控制层。

## 2. 这一阶段做了什么
这一阶段在主线工程的 `main.c` 里新增了一个轻量级 `VehicleControl` 骨架，并把它挂进了主循环。

当前控制方式采用“瞬时按键电机自检”：
- `KEY_1 / KEY_2` 控制 `motor A` 正反转
- `KEY_3 / KEY_4` 控制 `motor B` 正反转

同时保留了两条已经稳定的链路不变：
- `USART6` 接收来自 `F103` 的灰度运行态短帧
- OLED 持续显示灰度模块回传数据

为了让上板调试更直观，还加入了 LED 通道指示：
- `LED_1` 对应 `motor A` 是否有输出
- `LED_2` 对应 `motor B` 是否有输出

## 3. 这一阶段得到的直接结果
现在这条 F407/TB6612 主线已经具备了一个非常重要的能力：即使先不让车自动跑，也可以单独验证主控电机层是否正常。

这意味着接下来上板时，我们可以先回答这些最基础、也最关键的问题：
- `A/B` 两路到底各自接到了哪一侧车轮
- “正百分比”在实车上到底是前进还是后退
- 某一路电机到底有没有真正被主控拉起来

只要这几个问题没有确认，直接上自动寻迹会把排错难度拉得很高。

## 4. 这一阶段的工程状态
本轮修改后，主线工程已通过一次完整 Keil 命令行重建：
- 结果：`0 Error(s), 0 Warning(s)`

同时还顺手清理了 `motor_drive.c` 中一个旧的无意义比较告警，使这次阶段节点比进入本阶段前更干净。

## 5. 这一阶段留下的下一步
下一步最自然的动作已经很明确了：

1. 上板验证 `motor A / motor B` 与实车左右轮的对应关系。
2. 记录当前正负输出在整车上的真实前后方向。
3. 在确认这些基础映射之后，再把 `scene + turn` 接入左右轮差速控制。

换句话说，这一阶段完成的是“主控电机层从静态代码存在，推进到可上板人工验证”的关键过渡。

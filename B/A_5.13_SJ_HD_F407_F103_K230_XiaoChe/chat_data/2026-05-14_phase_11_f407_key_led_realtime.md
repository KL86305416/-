# 阶段摘要 11：F407 单键单灯实时自检
- 日期：2026-05-14
- 阶段编号：`11`
- 主题：`f407_key_led_realtime`
- 范围：`F407 main control panel / backups / dev_logs / chat_data`

## 1. 本阶段触发背景

在上一阶段完成了 `F407` 的最小按键点灯自检之后，用户进一步明确了希望看到的行为不是“按一下后锁存”，而是：

- 一个按键对一个灯
- 按下就亮
- 抬起就灭

这使得测试目标从“确认程序进没进去”进一步变成了：

- 更直观地验证按键输入和灯输出是否实时联动

## 2. 本阶段做出的调整

本阶段没有推翻前面的主控框架，只是把自检层从“锁存型”改成了“实时跟随型”：

- 删除边沿检测和锁存状态
- 保留主循环轮询方式
- 每轮直接读取 `PD5 / PD6`
- 立即驱动 `PB9 / PE0`

这样做以后，板上的现象会更贴近最原始的 GPIO 通断测试。

## 3. 当前一对一映射

本阶段采用的映射关系是：

- `D5 -> B9`
- `D6 -> E0`

也就是：

- `PD5` 控 `PB9`
- `PD6` 控 `PE0`

并按已经确认好的电气方向处理：

- 按键低电平按下
- LED 高电平点亮

## 4. 本阶段的意义

这一轮虽然改动很小，但意义很明确：

- 它把测试从“程序状态机是否被触发”收窄成了“输入输出通道是否实时有效”

如果这一步成立，就能非常直接地说明：

- 主循环在跑
- `PD5 / PD6` 输入没问题
- `PB9 / PE0` 输出没问题
- 板子上这组按键灯通路本身是活的

## 5. 当前上板预期

烧录这版固件后，最直观的预期表现是：

- 默认熄灯
- 按住 `D5`，`B9` 亮
- 松开 `D5`，`B9` 灭
- 按住 `D6`，`E0` 亮
- 松开 `D6`，`E0` 灭

## 6. 下一步最自然的动作

如果这一步实板行为正确，说明 `F407` 主控最基础的 GPIO 自检已经站稳。  
之后最自然的推进顺序就是：

1. 确认按键灯实时联动正常
2. 回到 `F103 -> F407` 串口短帧链路
3. 观察 `OLED` 是否同步显示灰度运行态
4. 再把 `turn/scene` 往电机控制层接

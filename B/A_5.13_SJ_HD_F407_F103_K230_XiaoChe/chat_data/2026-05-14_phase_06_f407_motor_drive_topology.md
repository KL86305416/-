# 阶段摘要 06：F407 电机驱动拓扑与 PWM 口分工澄清

- 日期：2026-05-14
- 阶段编号：`06`
- 主题：`f407_motor_drive_topology`
- 范围：`F407 main control panel / dev_logs / chat_data`

## 1. 本阶段目标

把 `F407` 主控侧“电机驱动方案到底是哪一套、8 路 PWM 到底是哪 8 路、舵机口和电机口怎么分开”这层基础关系先正式确认下来。

这是继续推进主控控制层之前必须先收口的一步。

## 2. 本阶段新增的关键信息

用户补充确认了主控电机部分的真实硬件准备：

1. 准备过两套驱动方案：
   - `DRV8874`
   - `TB6612`
2. 当前实际采用的是：
   - `DRV8874`
3. 当前准备了 `8` 路 PWM 输出口，用于 `DJ-XINX` 这一组电机控制线

这直接决定了后续软件抽象应该优先围绕“双 PWM 驱动一路电机”的模型来写。

## 3. 本阶段调研得出的结构结论

结合 `.ioc`、`tim.c` 和现有引脚定义，本阶段确认：

- 当前 8 路电机 PWM 线就是：
  - `DJ_A_IN1`
  - `DJ_A_IN2`
  - `DJ_B_IN1`
  - `DJ_B_IN2`
  - `DJ_C_IN1`
  - `DJ_C_IN2`
  - `DJ_D_IN1`
  - `DJ_D_IN2`

它们被分配在：
- `TIM1` 的 4 个通道
- `TIM2` 的 4 个通道

也就是说，当前硬件和代码结构都更像是：
- `4` 路电机
- 每路双 PWM
- 完全贴合 `DRV8874` 的当前使用方式

## 4. 本阶段同时澄清的另一个边界

这一轮还有一个很重要的澄清：

- `PB14 / PB15`
- `TIM12_CH1 / TIM12_CH2`
- `PWM_DuoJi_A / PWM_DuoJi_B`

这两路属于舵机 PWM，不属于上面的 8 路电机 PWM。

也就是说，从这个阶段开始，主控代码里要明确分成三类：
- 电机双 PWM
- 舵机 PWM
- 编码器反馈

## 5. 本阶段发现的一个风险点

在这轮核对里还发现：

- `TIM2` 虽然已经被配成 4 路 PWM 输出
- 但当前基础周期值还是 `0xFFFFFFFF`

这说明主控现在虽然“通道名义上齐了”，但“电机 PWM 基础参数”还没有真正进入可直接驱动的状态。

所以当前最稳的顺序仍然是：
- 先厘清驱动拓扑
- 再统一电机 PWM 基础参数
- 再写主控电机抽象层

## 6. 本阶段的意义

这一阶段虽然没有直接新增电机控制代码，但它把后续主控控制层最容易混乱的一层先理顺了：

- 现在到底按哪套驱动建模
- 哪 8 路才是电机 PWM
- 哪 2 路其实是舵机口
- 当前代码里哪个定时器配置还需要重整

把这层先定清楚，后面写 `motor_drive` 就不会一边写一边猜。

## 7. 本阶段之后最自然的下一步

在这一阶段之后，最自然的下一步是：

1. 统一 `TIM1 / TIM2` 的电机 PWM 基础参数
2. 新建一个只面向当前 `DRV8874` 的主控 `motor_drive` 层
3. 用“有符号速度/占空比”来封装每路双 PWM 电机
4. 等灰度短帧联调稳定后，再把 `TURN` 逐步挂到电机层上

这会是从“接口打通”走向“真正开始驱动主车体”的自然过渡。

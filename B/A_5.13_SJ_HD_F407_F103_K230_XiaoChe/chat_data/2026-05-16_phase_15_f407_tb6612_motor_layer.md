# 阶段摘要 15：F407 TB6612 分支电机控制层落地
- 日期：`2026-05-16`
- 阶段编号：`15`
- 主题：`f407_tb6612_motor_layer`
- 范围：`F407 main control panel TB6612 / backups / dev_logs / chat_data`

## 1. 这一阶段为什么开始

在确认 `TB6612` 分支已经能正常解析 `F103` 灰度短帧后，主控侧的下一块短板就从“看不到数据”切换成了“还不能可靠驱动电机”。

所以这一阶段的目标不是做寻迹闭环，而是先把电机底层补齐。

## 2. 这一阶段做了什么

这次真正落下去的核心内容有四块：

1. 新建 `TB6612` 专用 `motor_drive` 模块
2. 把 `PA15 / DJ_BIN2` 从模拟输入改成真正的方向输出口
3. 把 `TIM1 / TIM2` 调整成适合电机驱动的 PWM 参数
4. 在 `main.c` 里把电机层初始化正式接进主程序

这意味着 `TB6612` 分支第一次具备了完整的双电机底层控制骨架。

## 3. 当前这层已经具备的能力

当前电机层已经支持：

- 两路电机通道 `A/B`
- 驱动待机控制 `STBY`
- 正反转百分比控制
- 全部停止
- 全部短刹

它已经不再是单纯的 CubeMX 引脚工程，而是一个真正可供后续上层逻辑调用的驱动底层。

## 4. 这一阶段的意义

这一阶段的意义在于，把主控从“只能看灰度结果”推进成了“已经具备输出执行能力，但暂时还没接上完整策略”。

也就是说：

- 感知输入链已经通了
- 执行输出链也开始成型了

接下来就不需要再反复回到底层补引脚和 PWM，可以开始往“最小点机验证”和“左右轮映射确认”推进。

## 5. 当前最自然的下一步

下一步最合适的不是直接上完整寻迹，而是先做一个很小的电机动作验证：

1. 验证 `A/B` 分别对应哪一侧车轮
2. 验证正负百分比的实际方向
3. 验证停机、短刹是否符合预期

这一步跑通之后，主控就可以进入真正的整车联调阶段。

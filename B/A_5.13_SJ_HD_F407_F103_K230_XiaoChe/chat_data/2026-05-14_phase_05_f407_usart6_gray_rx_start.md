在 `F103` 已经切换到运行态短帧之后，正式把 `F407` 主控代码从“协议骨架阶段”推进到“真正能收灰度短帧”的阶段。

这一阶段的重点不是控制算法，而是先把主从之间的运行链条接起来。

## 2. 进入本阶段前的状态

在进入本阶段前，`F407` 工程已经具备：
- 基本外设初始化
- `gray_module` 骨架
- `vision_protocol` 骨架

但仍然存在明显空档：

也就是说，`F407` 还没有真正进入“联车态”。

## 3. 本阶段先调研再动手形成的结论

本阶段先做了完整调研，再决定代码落点，得出的关键结论是：

1. `USART6 (PC6/PC7)` 就是当前接 `F103` 智能灰度模块的主链路。
2. `UART4 (PA0/PA1)` 是给 `OpenMV` 预留的视觉链路，不应抢先成为第一优先级。
3. 当前 `gray_module` 还停留在旧版 `I2C / 长帧状态` 思路上，和 `F103` 当前运行态短帧已经脱节。
4. 因此本阶段最合理的第一步，不是碰电机和视觉融合，而是先把 `F103 -> F407` 的 `USART6` 运行态短帧接收链补完整。

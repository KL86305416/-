# 阶段摘要 07：F407 电机 PWM 基础收正与 OLED 灰度监视页接入

- 日期：2026-05-14
- 阶段编号：`07`
- 主题：`f407_motor_pwm_oled_monitor`
- 范围：`F407 main control panel / backups / dev_logs / chat_data`

## 1. 本阶段目标

在主控电机控制逻辑尚未完全接入之前，先完成两件对后续联调非常关键的基础工作：

1. 把当前 `DRV8874` 使用模式下的电机 PWM 基础参数收正
2. 在主控板 `OLED` 上直接显示 `F103` 传回来的运行态数据

这样做的目的很明确：
- 电机层不再停留在“口都分了但参数还不适合驱动”的状态
- 灰度链路联调不再只能靠外接串口

## 2. 进入本阶段前的状态

在进入本阶段前，`F407` 已经可以：
- 通过 `USART6` 接收 `F103` 的 6 字节短帧
- 在主控侧保存 `TURN / SCENE / FLAGS`

但仍然存在两个现实问题：

1. 电机 PWM 虽然通道都在，但 `TIM2` 的周期仍然不适合直接拿来做电机驱动
2. 主控虽然已经收到了灰度短帧，但板上还没有任何本地可视化窗口

## 3. 本阶段做出的关键代码动作

### 3.1 新建电机抽象层

本阶段新增了 `motor_drive` 模块，把当前实际使用的：
- `DRV8874`

先按“4 路电机、每路双 PWM”方式抽象出来。

这样主控代码层面已经具备：
- 统一初始化
- 统一停机
- 统一用有符号百分比描述电机方向与占空比

### 3.2 电机 PWM 基础参数收正

本阶段把电机相关 PWM 的基础周期收到了约 `20kHz`：

- `TIM1 -> ARR 8399`
- `TIM2 -> ARR 4199`

这使得电机 PWM 从“虽然分了口但参数不合理”，进入了“基础层已经可用”的状态。

### 3.3 默认安全策略

虽然电机 PWM 已经收正，但本阶段依然保持：
- 初始化后全部电机默认停机

这意味着本轮改动不会让底盘因为主控上电就自行动作。

### 3.4 新建 OLED 显示层与灰度监视层

本阶段新增：
- `oled_display`
- `gray_monitor`

其中优先走的是：
- `U5`
- `I2C1`
- `PB7 / PB8`

显示内容不是控制页面，而是灰度运行态监视页。

## 4. 本阶段建立起来的板上可视化能力

截至本阶段，`F407` 板上已经具备一个专门给灰度链路联调用的本地监视页。

它会周期显示：
- `F103` 是否在线
- 数据更新时间
- `TURN`
- `SCENE`
- `FLAGS`
- `LINE_FOUND / CENTER_ON_LINE`
- `LEFT / RIGHT / WIDE_BLACK`
- `UNSTABLE / SENSOR_FAULT / VALID`

这意味着：
- 就算电机控制层还没完全挂上
- 也已经可以先在主控板上看见 `F103` 是否在正确回数

## 5. 本阶段做出的一个明确工程假设

为了尽快建立可视化，本阶段做了一个明确而可调整的显示侧假设：

- 当前先按 `U5 I2C OLED`
- `SSD1306` 兼容类 `128x64`
- 地址尝试 `0x3C / 0x3D`

这不是说硬件一定只能这样，而是先用最可能成功的一条路径把链路搭起来。

如果板上实际屏参数略有差异，后面只需要在显示层做局部调整。

## 6. 本阶段的意义

这一阶段的意义很大，因为它让 `F407` 的主控调试从“后台接收短帧”变成了“板上就能看灰度运行态”。

从这个阶段开始：
- 主控已经不只是接收模块
- 也开始具备本地观察和联调支撑能力

这会显著降低后面接电机控制时的调试难度。

## 7. 本阶段之后最自然的下一步

完成这一阶段之后，最自然的下一步就是：

1. 上板确认 `U5 OLED` 是否点亮
2. 观察 `TURN / SCENE / FLAGS / online` 是否实时变化
3. 确认灰度链路稳定后
4. 再把 `TURN` 逐步接到底盘电机控制层

也就是说，这一阶段本质上是在为“真正开始主车体寻迹实跑”做最后一层准备。

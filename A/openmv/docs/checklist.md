# 比赛前检查清单

## 文件

- [ ] OpenMV 根目录存在最终版 `main.py`。
- [ ] `config.py` 已同步，阈值、ROI、串口参数是赛场最终值。
- [ ] `comm.py` 与 `protocol.py` 都已同步。
- [ ] 业务模块已同步：`camera.py`、`result.py`、`blob.py`、`line.py`、`shape.py`、`tag_code.py`、`filter.py`、`debug.py`、`guard.py`。
- [ ] 备用 `main_safe.py` 已保存在现场电脑和 OpenMV 盘符中。
- [ ] OpenMV 盘符中没有旧版重复脚本、临时备份或无关图片占空间。

## 硬件

- [ ] 摄像头排线牢固。
- [ ] 镜头焦距已调清楚。
- [ ] 摄像头安装角度和高度已固定。
- [ ] OpenMV 与主控共地。
- [ ] TX/RX 交叉连接正确。
- [ ] 电源稳定，断电重启后能自动运行。

## 图像

- [ ] 红、绿、蓝阈值在赛场灯光下稳定。
- [ ] `CAMERA_WHITEBAL`、曝光、增益设置符合当前光照。
- [ ] ROI 覆盖任务区域，不包含明显干扰物。
- [ ] 巡线模式能在直线、弯线、交叉干扰下保持输出。
- [ ] 形状、AprilTag、二维码模式在实际距离下能识别。
- [ ] IDE 帧缓冲区中的框选位置与实际目标一致。
- [ ] 正式运行时不依赖电脑端 IDE。

## 模式

- [ ] 默认 `DEFAULT_MODE` 符合比赛上电策略。
- [ ] `idle` 能切换并输出无目标结果。
- [ ] `red`、`green`、`blue` 均能稳定输出中心点。
- [ ] `line` 的 `angle` 偏移方向与主控约定一致。
- [ ] `shape`、`apriltag`、`qrcode` 的 `payload` 或 `id` 与主控解析一致。
- [ ] `auto` 的优先级符合任务要求。
- [ ] 别名 `tag`、`april`、`qr`、`code`、`color` 不会让主控误触发。

## 串口

- [ ] UART 端口为 `3`，波特率为 `115200`，或主控已同步修改。
- [ ] 主控能发送纯文本 `mode:*` 调试命令。
- [ ] 主控能发送 `$MC,MODE,*XX` 和 `$MC,STOP*3A` 协议帧。
- [ ] 主控解析 `$MV,RESULT,...*XX` 字段顺序与 `docs/protocol.md` 一致。
- [ ] 连续运行 3 分钟无断流、无乱码、无校验错误。
- [ ] 丢目标时 `ok=0`、`miss` 递增或保持逻辑符合主控预期。

## 应急

- [ ] `main_safe.py` 可独立运行。
- [ ] 红色色块救援输出 `1,x,y` / `0,0,0`。
- [ ] 现场电脑保留完整工程副本。
- [ ] 记录最终阈值、ROI、曝光、增益、串口参数。
- [ ] 比赛前不再随意改动已验证参数。
- [ ] 出现异常时先切 `main_safe.py`，再排查主工程。

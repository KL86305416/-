# 项目日志：丢线兜底也先前进并降低直角转向力度

- 时间：2026-05-13
- 改动范围：直角弯丢线入口、转向速度、转后稳定期

## 问题现象

- 小车识别到直角后没有明显先前进一段，仍然像之前一样直接开始转。
- 转弯动作偏大，转完后容易再次读到黑线并触发找线反打。

## 原因分析

- 正常识别路径已经会进入 `ENTRY` 前进阶段，但“刚丢线兜底”路径仍然直接进入 `PIVOT`。
- 实车上很多直角弯触发点来自丢线兜底，因此看起来仍然没有先前进。
- `PIVOT` 的外侧轮和内侧反转速度偏强，转向角速度过大。

## 改动内容

- 将丢线兜底入口从直接 `PIVOT` 改为先进入 `ENTRY`：
  - 新增 `LineFollower_StartRightAngleAfterLost()`；
  - 丢线后也先低速前进，再进入转向阶段。
- 延长直角弯前进阶段：
  - `LINE_FOLLOWER_RIGHT_ANGLE_ENTRY_MS = 70U`；
  - `LINE_FOLLOWER_RIGHT_ANGLE_ENTRY_SPEED_DUTY = 95`；
  - 前进阶段差速仍保持 `0`，尽量直走。
- 降低直角转向力度：
  - 外侧轮速度 `145 -> 115`；
  - 内侧轮反转 `-85 -> -35`；
  - 最小转向时间 `80 ms -> 60 ms`；
  - 强制退出时间 `280 ms -> 240 ms`。
- 加强转后稳定期：
  - 稳定期 `180 ms -> 320 ms`；
  - 稳定期速度 `90 -> 80`；
  - 稳定期最大修正 `45.0f -> 28.0f`；
  - 稳定期每周期修正步进 `3.5f -> 2.0f`。

## 验证结果

- 已执行 `cmake --build build/Debug`，编译和链接通过。
- 构建结果：RAM 使用 `3328 B / 32 KB`，FLASH 使用 `25492 B / 128 KB`。

## 调参建议

- 如果仍然转得过猛，优先把 `LINE_FOLLOWER_RIGHT_ANGLE_INNER_SPEED_DUTY` 调到 `0`，让内侧轮不反转。
- 如果转弯不够，再把 `LINE_FOLLOWER_RIGHT_ANGLE_OUTER_SPEED_DUTY` 从 `115` 逐步调到 `125` 或 `135`。
- 如果仍然没有明显前进阶段，把 `LINE_FOLLOWER_RIGHT_ANGLE_ENTRY_MS` 调到 `90U`。
